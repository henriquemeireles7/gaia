# Initiative 0002 — Gaia Template Bootstrap

## 1. Context / Research

**Today's state (post-0001).** The methodology refactor shipped on master (`c0c7272`). 18 skills exist with 1:1 reference pairing. Fractal `CLAUDE.md` covers every domain folder. `apps/api` (Elysia), `apps/web` (SolidStart), `packages/*` (auth, db, security, adapters, ui, errors, config, core, workflows) are scaffolded. Auth + Polar checkout end-to-end landed in `6da6c31`. UI tokens + components landed. CI gates (oxlint, tsgo, knip, security scans) are green. The only thing missing for a developer to "clone and ship" is **the front door** — a CLI that closes the loop from `git clone` to live URL with continuous narration.

**Carry-overs from the archived research** (`_archive/2026-04-27-gaia-bootstrap-source.md`, 2026-04-27, 38KB). The cap table for template surfaces, the 30-minute wall-time analysis, and the deploy-self-heal magical moment all carry forward. What does **not** carry forward (per 0001 F-6): the `/d-onboard` orchestrator skill — onboarding is now CLI-verbs-only.

**Adjacent products.** ShipFast / Makerkit / Supastarter sell paid Next.js + Supabase + Stripe scaffolds with 2-6 hour TTFD ceilings. T3 / RedwoodSDK / Wasp / Encore.ts ship the framework but not the SaaS surface. **None ship an agent-native CLI with live narration**. Gaia's wedge is that the CLI is a typed, NDJSON-streaming, agent-readable substrate AND the human reads three sentences in the README and then watches the terminal.

**Pre-launch context.** Zero users. No backwards compatibility. Founder is the only alpha tester per Q6=A — but post-launch anonymized first-run telemetry will confirm or falsify the claim across external users (founder authorized telemetry expansion 2026-04-29 during autoplan).

## 2. Strategy

**Problem.** A complete template that doesn't ship with a working CLI is a half-finished promise. Without `bun create gaia-app@latest <name>`, there is no path that lets a developer evaluate Gaia in a Saturday afternoon. With it, the template's architectural advantages become legible in the first 30 minutes — and the magical moment isn't minute 28, it's _second 0.5_ when the banner appears.

**Hypothesis** (echoes frontmatter). Agent-native CLI + complete template + continuous live narration = sub-30-minute clone-to-deploy on first run.

**Narrowest wedge.** A 60-second README demo showing: terminal opens, `bun create gaia-app@latest weekend-saas` runs, banner appears at second 0.5, timestamped event stream rolls for ~28 minutes, deploy fails once, `d-fail` self-heals, smoke-test prints celebration banner with live URL.

**Approach (chosen): founder spec + full agent surface + full docs + minimal template completeness gate + founder-only launch + post-launch telemetry.**

**Reframings adopted (within locked scope):**

1. **Magical moment is second 0.5, not minute 28.** Banner appears at TTHW-1 (<1000ms target) with project slug + version + "30-MIN CLOCK STARTED". README demo GIF leads with second 0.5.
2. **README is PR 1 (scaffold + claims-to-retire) AND PR 10 (book-end rewrite from real run).** PR 1 ships before verbs exist; PR 10 rewrites from actual founder timing so launch README isn't aspirational.
3. **`state.json` IS the resume primitive.** Every verb resumable from last checkpoint. Without this, `d-fail` fixing one class still forces full re-run. Schema locked NOW (TypeBox v1) since file crosses public-flip boundary.

**Abandonment ladder.**

- TTHW-1 banner > 1000ms on clean machine: investigate Bun cold-start; if unfixable, accept ≤2000ms.
- Founder's first clean-machine run > 45 min: re-scope (likely strip Q2=C agent surface to Q2=A minimum + drop docs site stub).
- Founder's first run > 60 min: cut `bun gaia explain` (PR 7) and `docs/cli.md` auto-gen — verbs ship with `--json` minimum.
- d-fail fails on a class outside the 7-class taxonomy: extend to 5-attempt cap; expand failure-class catalog only if 2+ classes are missed in a single run.
- Post-launch telemetry shows <50% of external first-runs reach smoke-green within 60 min: hold v1.1 launch, run `/d-fail` audit on the 0003 backlog.

## 3. Implementation

**Order of operations** (so nothing breaks mid-build):

1. README rewrite + GitHub repo metadata + .github/ templates + OG image + badges (PR 1) — every later PR references it.
2. CLI package skeleton + `bun create` scaffolder + Bun version preflight + Windows refusal + dir-exists semantics + npm name reservation week 1 + telemetry posture stated + .gitignore-FIRST ordering invariant (PR 2). Banner + TTHW-1 benchmark gate.
3. Typed protocol (TYPE-ONLY) + agent-native primitives + telemetry emitter + live narration default-on + standard flag set + POSIX exit codes (PR 3). `cli.ts` schema, `events.schema.ts`, `state.schema.ts` (TypeBox v1).
4. Three onboard verbs (PRs 4-6) in dependency order: `verify-keys` (read-only) → `deploy` (d-fail subprocess + --with-ci flag for GH Actions secrets sync) → `smoke` (post-deploy assertions including Polar webhook round-trip + auth cookie posture).
5. `bun gaia explain` + error catalog extending `packages/errors/` (E1xxx-E4xxx, min 12 codes, 4 fields each) + `next:` hint convention across all verbs (PR 7).
6. Template completeness E2E gate — fresh-clone CI matrix (ubuntu-latest + macos-latest, 6 assertions, 12-min budget per matrix entry) + `.env.example` parity gate + dashboard "minimal but designed" cap (PR 8).
7. Docs surface — README PR 1 polish, `docs/getting-started.md` (7-section locked outline), `docs/cli.md` (auto-gen + CI drift detect), `docs/architecture.md` (agent-tagged), `docs/contributing.md` (references `/d-code` + manual upgrade story), `docs/privacy.md` (telemetry), Scalar docs site stub (PR 9).
8. 60-sec demo recording + `docs/launch.md` + README book-end rewrite from real founder run timing (PR 10).
9. v1.0 tag + public repo flip (PR 11).

**Risks (priority-ordered).**

1. **Q2=C scope creep.** Typed protocol + NDJSON + state.json + explain + next-hints + telemetry + flag set + exit codes is sizable. _Mitigation:_ PR 3 ships the foundation; verbs inherit. If timeline slips, defer `explain` (PR 7) to 0003; verbs still ship with `--json`.
2. **`state.json` schema drift.** File crosses public-flip boundary; later initiatives inherit the contract. _Mitigation:_ Lock TypeBox schema in PR 3 with `version: 1` field and migration story documented in `docs/architecture.md`.
3. **Founder-clean-machine isn't actually clean.** _Mitigation:_ Tightened falsifier — macOS guest user account, no Brew, fresh Bun via curl, fresh provider accounts. Plus post-launch telemetry as second signal.
4. **CLI npm publish is irreversible.** Once `gaia` is published the unscoped name is locked. _Mitigation:_ Reserve `gaia` on npm in PR 2 (week 1) with placeholder package; publish real CLI as `@gaia/cli`; promote unscoped at v1.0. Squat risk closed in week 1.
5. **Docs drift.** `docs/cli.md` auto-generated from `--help` breaks if `--help` text changes silently. _Mitigation:_ CI step regenerates `docs/cli.md` and fails on diff (PR 9).
6. **Dashboard "skeleton" cap risks shipping a wireframe.** _Mitigation:_ Re-cap'd as "minimal but designed" — auth-gated landing + account/billing tabs with designed empty/loading/error states using existing `apps/web/src/components/empty-state.tsx` + `packages/ui` tokens. Mobile responsive QA gated in PR 8.
7. **d-fail loops infinitely on unmapped class.** _Mitigation:_ 3-attempt hard cap with exponential backoff; surfaced-cleanly catch-all path prints actionable text + `bun gaia explain E5000`.
8. **Hook layer accidentally imports from `cli/`.** _Mitigation:_ `.gaia/protocols/cli.ts` is type-only by lint rule; runtime stays in `cli/src/`.
9. **Telemetry leaks PII.** _Mitigation:_ Schema-validated event payloads with explicit allowlist in `cli/src/telemetry.ts`; PostHog `disable_geoip: true` and no `$ip` field; CI test asserts no PII in synthetic event payloads.
10. **Half-clean machine masks cold-start failure modes.** _Mitigation:_ Post-launch `cli.first_run` telemetry detects external user failure rates within 14-day window.

**Out-of-scope (explicit, reaffirmed).** Admin v0.1, content v0.1, AI demo feature, external alpha sessions, Show HN/X thread, multi-tenancy, MFA/SSO, `bun gaia upgrade`, `bun gaia generate`, conductor.build integration, Windows support (refused with actionable message until v1.1), behavioral telemetry beyond first-run anonymized events, broader d-fail classes beyond the 7 listed (DNS, full Railway quota → defer to 0003).
